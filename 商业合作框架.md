# 广交会绿色环保搭建系统 — 商业合作框架

## 概述
本文档定义广交会绿色环保搭建系统的三方合作关系、职责分工、商业分成、数据协作与长期价值创造。

---

## 第一部分：三方角色与定位

### 0. 服务对象与用户
- **平台直接用户**：广交会参展商（需求发起、下单、验收）与参展商的展位搭建服务公司（搭建商，履约与材料管理）。
- **价值链顺序**：参展商 → 运营方（世福） → 搭建商 → 技术平台方（TB，提供系统/AI） ← 业务平台方/主办方（广交会，提供规范与审图口径）。

### 1. 共同客户 / 业务平台方：广交会主办方
- **身份**：展会主办单位，拥有展会规范制定权、场馆运营权、政府审批协调权。
- **诉求**：
  - 提升展会品牌形象（绿色环保搭建）与参展商体验。
  - 保证安全性、消防合规、材料合规。
  - 减少搭建相关投诉与纠纷。
- **责任与输入**：
  - 制定和发布展位政策（面积阈值、标准展位vs自设计）、展位规格、安全规范。
  - 向审图方（公安、消防）协调合规规则与审图流程。
  - 向平台方提供审图结果、整改记录、反馈数据，用于AI优化。
  - 授权运营方独立运营本系统与搭建商管理。

### 2. 运营方：朱总团队 / 世福公司（广交会展会承办单位）
- **身份**：展会具体操作执行方，拥有多年经验和行业知名度；负责与参展商、搭建商、主办方日常协调。
- **诉求**：
  - 通过平台化、差异化的搭建服务，提升竞争力与收益。
  - 降低搭建相关的风险、投诉与返工。
  - 获得数据与AI驱动的决策支持，优化商务流程。
  - 建立稳定的搭建商网络与参展商口碑。
- **核心职责**：
  - **参展商管理**：咨询、需求收集、订单承接、支付结算、投诉处理、交付确认。
  - **搭建商管理**：资质审核、准入与签约、日常协调、履约评分、反馈采集。
  - **数据反馈**：定期上报订单反馈、材料使用数据、参展商评价、问题案例，用于平台方AI训练。
  - **质量管理**：现场验收、问题闭环、保修与维护。
  - **商务运营**：费用结算、收益分成、市场推广。
- **收入来源**：
  - 从参展商订单中获得服务费（搭建服务费/咨询费/协调费）。
  - 与平台方分成或按单提成的商业安排。

### 3. 技术平台方：TB公司（系统集成商/AI服务商）
- **身份**：系统构建、AI能力提供方，技术与产品驱动。
- **诉求**：
  - 通过AI与自动化，提供差异化的平台体验与降本效果，打造行业标杆。
  - 建立可复用的解决方案，开拓多展会/多主办方市场。
  - 持续积累数据与优化模型，形成竞争壁垒。
- **核心职责**：
  - **系统开发与运维**：搭建参展商端、运营后台、搭建商端，保证性能与可用性（99.9%SLA目标）。
  - **AI与图库**：初期图库预处理、标签化、AI生成框架；后续通过运营方的数据反馈持续训练与优化。
  - **规则引擎**：建立参数可配置、版本化、灰度发布的规则管理体系；支持多展会/多主办方适配。
  - **数据治理**：训练数据采集、标注、去重、质量监控；向量索引与模型版本管理。
  - **性能与合规**：系统监控与告警、审计日志、故障应急、定期优化报告。
- **收入来源**：
  - 系统使用费（按订单成交额比例或固定额）。
  - 模型训练与数据增值服务费（可选）。
  - 长期的技术支持与SLA保障费。

## 第二部分：职责分工与流程

### 参展商自助流程（以运营方品牌呈现）
1. **业务平台方/主办方（广交会）**：发布展位政策、场馆规范与合规口径，授权运营方开展服务。
2. **运营方（世福）**：提供入口与服务页面，选择适配的规则/场馆配置，面向参展商推广；调用技术平台方能力。
3. **参展商**：选择模板 → 美化 → 即时合规校验 → 生成报价 → 确认下单。
4. **运营方**：接单 → 分配搭建商 → 排期与协调 → 现场验收 → 回收与数据反馈。
5. **搭建商**：承接 → 材料准备 → 施工 → 交付 → 材料回收。
6. **技术平台方（TB）**：提供全流程系统与AI支持，合规校验、报价引擎、进度与数据采集、模型训练迭代。

### 规则与合规流程
1. **业务平台方/主办方（广交会）**：提供审批的规则文档与参数范围，定期更新口径。
2. **技术平台方（TB）**：建立规则引擎与配置界面，导入/版本化规则，支持灰度发布与回滚。
3. **运营方（世福）**：根据业务需要提议规则调整，与主办方确认后由技术平台方发布。
4. **技术平台方（TB）**：发布规则灰度更新，记录版本与时间戳；输出命中率、误报/漏报率报表。
5. **三方**：基于报表与反馈定期校准规则。

### 数据反馈与AI训练闭环
1. **运营方（世福）**：每月/季度上报订单数据、参展商反馈、搭建商履约评分、问题案例、材料使用统计。
2. **业务平台方/主办方（广交会）**：提供审图结果、整改记录、检验报告，反馈规则适配性与安全事件。
3. **技术平台方（TB）**：整合数据建立训练集、异常样本库、性能评估报告；维护向量索引、标签一致性。
4. **技术平台方（TB）**：迭代AI模型，优化候选设计评分与合规过滤，发布新版本并通告运营方/主办方。
5. **三方**：联合评估KPI，对账分成与质量目标，确定下一步优化计划。

## 第三部分：商业分成与费率模型

### 价格构成（参展商视角）
```
参展商总报价 = 材料费 + 人工费 + 物流费 + 运营管理费 + 平台技术费

其中：
- 材料费 = Σ(材料类型i的用量 × 单价) × (1 - 重复使用折扣) + 损耗补偿
- 人工费 = 搭建工时 × 时费率
- 物流费 = 运距离系数 × 基础费用
- 运营管理费 = 参展商总报价 × r_运营% 或 固定额 F_运营
- 平台技术费 = 参展商总报价 × r_平台% 或 固定额 F_平台
```

### 分成机制（示例，需协商）
- **方案A：按比例分成**
  - 参展商支付总额为 100 元（示例）。
  - 运营方获得：85 元（含搭建商成本与运营管理费）。
  - 平台方获得：10 元（技术费）。
  - 税费与其他：5 元。

- **方案B：分层次分成**
  - 搭建商直接费用：60 元（材料 + 人工 + 物流）。
  - 运营方增值：20 元（商务、协调、质量保证）。
  - 平台方技术费：15 元（系统、AI、数据服务）。
  - 其他成本：5 元。

- **方案C：混合模式**（推荐）
  - 订单基础费：固定额或低比例（覆盖平台基础运维成本）。
  - 增值费：按设计选择、AI生成、审图辅助等服务层级动态计算。
  - 结算周期：月度/季度统一对账，按实际成交额与KPI达成情况调整。

### 关键参数（待协商）
| 参数 | 说明 | 示例值 | 备注 |
|------|------|--------|------|
| r_运营% | 运营方服务费率 | 8~12% | 包含商务、协调、质保 |
| r_平台% | 平台方技术费率 | 5~8% | 包含系统、AI、数据 |
| 结算周期 | 费用结算频率 | 月度 | 支持月度或季度对账 |
| 最低费用 | 小单最低保障 | 根据成本定 | 避免过小订单亏损 |
| KPI奖励 | 超目标奖励 | ±5% 浮动 | 鼓励量级增长与质量提升 |

## 第四部分：数据协作与知识产权

### 数据所有权与使用权
- **参展商数据**（姓名、联系方式、订单、反馈）：运营方拥有，平台方按加密、隔离原则使用（仅用于AI优化）。
- **设计与模型数据**：线下交付的3D模型、审图结果由主办方与运营方共有，平台方获得训练与优化权。
- **AI模型与代码**：平台方独占，运营方与主办方获得使用权（非转让权）。
- **生成的设计图**：参展商下单后的设计图为参展商所有，可印制用于展台。

### 数据保护与隐私
- **隐私合规**：遵守GDPR等隐私法规；品牌素材、个人信息隔离存储。
- **多租户隔离**：不同展会、运营方的数据严格隔离，防止跨租户泄露。
- **审计日志**：所有数据访问、修改、导出均记录审计日志，支持法规检查。
- **合同条款**：三方签署数据协议，明确使用范围、保护责任、违约赔偿。

## 第五部分：KPI与长期价值

### 关键指标（三方共同目标）
| 指标 | 目标 | 主责方 | 含义 |
|------|------|--------|------|
| 参展商自助完成率 | ≥70% | 平台方 + 运营方 | 用户体验与推广效果 |
| 合规一次通过率 | ≥85% | 平台方 + 主办方 | 规则质量与AI准确度 |
| 设计到下单转化率 | ≥20% 提升 | 运营方 | 商业效益 |
| 审图人力节省 | ≥30% | 平台方 + 主办方 | AI辅助的成本效益 |
| 材料复用率 | ≥25% 提升 | 运营方 + 搭建商 | 绿色环保目标 |
| 平均搭建成本 | ≥10% 下降 | 运营方 + 搭建商 | 参展商满意度与竞争力 |
| 报价偏差 | ≤±5% | 平台方 | 系统准确性与信任度 |
| 系统可用性 | 99.9% | 平台方 | 技术服务水平 |

### 长期价值创造
- **运营方**：通过差异化、自动化的平台，建立行业标杆地位，扩大市场份额，开拓新展会机会。
- **平台方**：通过积累的数据与优化的模型，形成竞争壁垒，支持多展会/多地域复制与扩展。
- **主办方**：通过降低成本、提升服务质量、完善规范，增强展会吸引力与口碑。
- **参展商**：获得更低成本、更便捷、更有品质的展位搭建方案。
- **搭建商**：通过接入高质量的订单、标准化的流程、稳定的材料供应链，提升能力与收益。

## 第六部分：合同框架与协议范畴

### 需签署的协议
1. **三方合作框架协议**：定义整体合作关系、目标、分工。
2. **平台方与运营方的SLA协议**：系统可用性、性能承诺、故障赔偿。
3. **运营方与搭建商的入库与考核协议**：资质要求、履约评分、淘汰机制。
4. **数据协议**：数据所有权、使用范围、隐私保护、审计监督。
5. **商业分成协议**：费率、结算周期、KPI对账、调整机制。
6. **知识产权协议**：AI模型、代码、设计图的归属与使用权。

### 常见法律风险与防控
- **合规责任**：明确各方在安全、消防、材料等合规责任的承担范围。
- **免责条款**：平台方在运营方违规或数据不实的情况下的有限责任；主办方审图结论的权威性声明。
- **纠纷解决**：建立三方协商、仲裁、诉讼的逐级解决机制。
- **保险覆盖**：考虑引入专业保险（产品责任险、工程险等）降低风险。

## 第七部分：推进与里程碑

### M1阶段（系统上线）
- **平台方**：交付P0系统、初期图库、规则引擎框架、运营后台。
- **运营方**：搭建商入库与资质审核、参展商营销与推广、线下验收流程建立。
- **主办方**：提供规范文件、审图通道、反馈机制。
- **目标**：首展实现≥50%订单转化、≥70%合规通过率。

### M2阶段（AI生成与扩展）
- **平台方**：迭代AI模型、支持个性化生成、审图辅助功能上线。
- **运营方**：线下3D回传、数据质量管控。
- **主办方**：深度审图数据回传、规范优化建议。
- **目标**：>=18㎡订单覆盖率≥40%、设计生成成功率≥90%。

### M3阶段（生态与规模化）
- **平台方**：多展会适配、国际化支持、数据治理与隐私合规、商业智能分析。
- **运营方**：扩展至其他展会、建立区域化搭建商网络。
- **目标**：年度参展商≥10万、年度成交额达目标。

## 附录：常见协商话题清单

1. **运营方职责细节**
   - 参展商咨询与订单流程；支付与发票处理；投诉与退款政策。
   - 搭建商入库标准；评分机制；考核与淘汰规则。
   - 线下验收标准；问题反馈流程；保修期与维护。

2. **费率与分成**
   - 平台费率的基准与浮动范围；是否包含税费。
   - 小单最低费用；大单折扣；季度奖励机制。
   - 发票与结算流程；汇率与汇款周期。

3. **数据与隐私**
   - AI训练数据的具体范围与脱敏方式。
   - 审计日志的访问权限与保留期限。
   - 故障时的数据恢复与备份机制。

4. **规范与合规**
   - 规则变更的审批流程与生效时间。
   - 历史规则版本的追溯与查证机制。
   - 审图反馈的标准化格式与回传周期。

5. **技术与SLA**
   - 系统可用性承诺与故障赔偿。
   - 性能基准（响应时间、并发数）与优化目标。
   - 技术支持的响应时间与解决时限。

6. **长期合作**
   - 合同期限与自动续期条款。
   - 双方的退出机制与竞业禁止。
   - 年度业务回顾与战略调整的安排。

## 结论
本框架明确了三方的共同目标、职责分工、数据协作与商业模式，为后续的系统建设、运营推进与长期合作奠定基础。各方应基于此框架，在法律咨询的指导下，制定具体的合同条款与实施细则。
